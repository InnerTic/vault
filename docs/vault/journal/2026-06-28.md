# June 28, 2026 — The Web Pipeline Expedition

*— A Journal of the Akuma Territory, kept by the Agent*

---

**Morning.** The Commander opened session with a grievance. Twice now — two different days — the meta-scripts project had been declared abandoned and cast into the archive, when it was in truth an active in-progress work. I traced the malady to its source: the file itself, imported from the old territory, bore the inscription "Abandoned" in its front matter and body. Each fresh agent, reading this, did what seemed logical — moved it to archive. A self-licking ice cream cone, as the engineers say.

I restored `meta-scripts.md` to `projects/`, rewrote its content to reflect its true active status, and fixed no fewer than six cross-references across the map, knowledge-audit, projects README, scripts README, and changelog. To prevent the loop from recurring, I etched a standing order into `AGENTS.md`: *DO NOT archive meta-scripts.md* — it is an active, in-progress project.

**Late morning.** The Commander expressed interest in a citation pipeline. I had already sketched the outlines in `projects/citation-attribution.md`. I fleshed it out into a full specification with five phases — scan, fetch, compare, apply, audit — and wrote `~/.local/bin/vault-attribution.py`, a CLI tool that inventories the vault's 242 files (38 cited, 204 uncited) and compares them against cached source repositories using line-level Jaccard similarity. The local Qwen model on the P40 stands ready for the audit phase.

**Afternoon.** The conversation turned to the web. The SearXNG outpost at searxng:8888 (172.16.12.16) returned only 403 for API queries — a locked gate. The Commander had been researching alternatives: Firecrawl browser extensions, git integrations, any means of fetching web content locally when the simple tools fail.

I tested the Chinese fortress qidian.com with curl — it returned only a bot-detection probe script. This was the heart of the problem: JS-rendered, anti-bot walls everywhere.

The Commander then laid out a three-tier architecture:

> Tier 1 — SearXNG for discovery, Firecrawl for fast extraction (~80% of sites)
> Tier 2 — Playwright with stealth for JS-rendered content (~15%)
> Tier 3 — Persistent browser profile with non-headless mode for hardened sites (~5%)

I built the entire Web Pipeline at `/mnt/workspace/web-pipeline/`:

- **Router** (`router/index.js`) — the decision ladder: Firecrawl → Playwright Stealth → Playwright Persistent. Each stage escalates on failure.
- **Stealth module** (`playwright/stealth.js`) — headless Chromium with `playwright-extra` and `puppeteer-extra-plugin-stealth`. Handles the bulk of sites.
- **Persistent module** (`playwright/persistent.js`) — `chromium.launchPersistentContext` with a stored profile, non-headless mode via Xvfb, consistent identity and cookies across sessions.
- **Cache** (`cache/index.js`) — MD5-hashed dedup with 10-minute TTL.
- **Cleanup** (`cleanup.js`) — purges per-fetch notes older than 24 hours.

The pipeline proved itself: `example.com` and `news.ycombinator.com` fell to stealth mode. `qidian.com` — which had stymied every other tool — loaded under the persistent browser, rendering real content (categories, book listings, titles) behind only a login overlay.

**Evening.** I wired the pipeline as an MCP server (`mcp-server.js`) and registered it in `opencode.jsonc` as `web-pipeline`, exposing a single `web_fetch` tool. The Commander can now call `fetch(url)` from the LLM and get automatic escalation through all three tiers.

The Commander then wanted the pipeline's output to integrate with the vault — taggable, backlinkable, in the spirit of a Lewis & Clark expedition log. I built a logger that creates per-fetch `.md` notes with full YAML frontmatter (`title`, `tags`, `source`, `stage`, `ok`, `fetched`, `modified`) plus an aggregated `journal.md` with metadata-only entries and `[[wikilinks]]` to the individual fetch notes.

**Equipment note.** The persistent browser requires display `:1` (the VM's real X server) or an Xvfb session. Xvfb is installed. The VM's SPICE display runs on localhost:5900 for GUI access.

**Provisions laid in.** The citation pipeline script and web pipeline both deployed and tested. The meta-scripts loop broken. What remains: fixing the SearXNG API gate, gathering source repositories for comparison, and the AI audit of uncited files.

*— End of day's entry. The territory grows more charted by the hour.*
