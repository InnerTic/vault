# 20260628-003 — Web Pipeline

## Goal

Reliable local web acquisition that handles JS-heavy and anti-bot sites.

## Architecture

3-tier escalation pipeline at `/mnt/workspace/web-pipeline/`:

```
Firecrawl (fast path)
    ↓ fail
Playwright Stealth (playwright-extra + stealth plugin)
    ↓ fail
Playwright Persistent (chromium.launchPersistentContext + Xvfb)
    ↓ fail
(optional proxy retry)
```

### Components

| Module | Path | Role |
|--------|------|------|
| Router | `router/index.js` | Decision ladder, stage escalation |
| Stealth | `playwright/stealth.js` | Headless + anti-detection (80% of sites) |
| Persistent | `playwright/persistent.js` | Non-headless, stored profile, `--force-dark-mode` |
| Cache | `cache/index.js` | MD5-hashed, 10-min TTL |
| Logger | `logger/index.js` | Per-fetch `.md` notes + aggregated journal |
| MCP | `mcp-server.js` | Exposes `web_fetch` tool to LLM |
| Cleanup | `cleanup.js` | Purges fetch notes >24h |

### Pipeline routing

```js
function fetch(url):
    result = firecrawl(url)
    if result.ok: return result

    result = playwright_stealth(url)
    if result.ok: return result

    result = playwright_persistent(url)
    return result
```

### Failure detection

- Empty markdown, "enable javascript", "access denied", HTML < threshold → Firecrawl failure
- Page title contains "captcha", "just a moment", "verify you are human" → Playwright failure
- DOM contains cloudflare, datadome, perimeterx → Playwright failure

## Decisions

**Persistent browser:** `chromium.launchPersistentContext` with profile at `profiles/default/`. Non-headless via Xvfb. `--force-dark-mode` for comfortable viewing.

**Display:** Xvfb on `:99` for automated runs. VM's real X server on `:1` for interactive use. SPICE on localhost:5900.

**Cache key:** `MD5(url + mode)` with 10-min TTL to avoid repeated bot triggers.

**Logging:** Per-fetch `.md` files with YAML frontmatter (`title`, `tags`, `source`, `stage`, `ok`). Aggregated `journal.md` with `[[wikilinks]]` to fetch notes. No raw page content in logs — content stays in cache.

**MCP:** Single `web_fetch` tool with `url`, `mode` (auto/firecrawl/stealth/persistent), and `raw` (return HTML) parameters. LLM never sees complexity.

## Problems

### SearXNG API returns 403

Still unresolved. Web UI works at searxng:8888. API queries return Forbidden. Needs `search_token` or `limits.exception` config on the LXC — cannot SSH in (no key).

### qidian.com blocks headless

curl returns empty bot probe page. Headless Chromium times out with redirect loop. Persistent browser (non-headless + profile) renders real content — categories, book listings, titles — behind only a login overlay.

## Discoveries

- qidian.com blocks curl (empty probe page)
- qidian.com blocks headless Chromium (redirect loop)
- Persistent browser with stored profile succeeds where headless fails
- Playwright + stealth plugin handles example.com and HN without issues
- Xvfb + `launchPersistentContext` works reliably for rendering JS-heavy pages

## Lessons

- A graduated escalation stack costs less than a single heavy tool for all cases
- Persistent browser profiles defeat JS-rendered anti-bot walls where headless fails
- Three tiers (~80/15/5 split) covers the vast majority of real-world sites
